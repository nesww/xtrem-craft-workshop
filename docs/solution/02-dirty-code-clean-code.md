# Clean code VS Dirty code

## Qu'est-ce que du code sale ?

- Dirty code is non-maintainable code, hardly testable, with strong coupling between parts.
- Business is also all around, not S.O.L.I.D
- No clear conventions (bad names, camel-case vs kebab-case)
- bad conventions in git repository (no commit conventions)
- Name not clear
- Comments
- Long method
- Duplicated code

## Qu'est-ce que du code propre ?

- Easily maintainable
- Business naming
- Standardized syntax
- SRP
- Isolated dependencies (design patterns ->  i.e. adaptator)
- Automated processes
- Defined practices (SOLID, KISS, CUPID, YAGNI, DRY)
